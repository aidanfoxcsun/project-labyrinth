# User Guide — Project Labyrinth
## Who This Guide Is For

This guide is intended for instructors, TAs, and new users who want to run and test Project Labyrinth in Unity.

## System Overview

Project Labyrinth is a 2D top-down roguelike game with procedurally generated dungeon floors (for this guide)
Each floor is generated on a grid, assigns special room types (Start, Boss, Treasure, Upgrade), connects rooms with doors, and spawns enemies, rocks, and other room content.

A new dungeon layout is generated automatically when the scene starts, and new floors can be generated during gameplay.

## Installation & Setup
Unity (use the version specified in the repository)

Git or GitHub Desktop

Clone the repository:

    git clone <your-repo-url>

Open Unity Hub.

Click Add Project and select the root project folder (the folder containing Assets, Packages, and ProjectSettings).

Open the project.

Open the main scene used for gameplay.

## How To Run The Project

Press Play in Unity.

The dungeon will generate automatically at runtime.

Verify that:

    A Start room exists.

    A Boss room exists and is placed far from Start.

    A Treasure room exists.

    An Upgrade room exists.

    Normal rooms contain enemies and rocks.

## Floor Progression

After defeating the boss:

    Enter the portal (if implemented in your project).

The next floor is generated by calling:

    DungeonGenerator.GenerateNewFloor()

The player should be repositioned to the new Start room.

## Outputs & What To Expect

Rooms are instantiated under the DungeonGenerator object in the Hierarchy.

    Special rooms are visually distinct (based on color coding).

    Normal rooms contain enemies and rock obstacles.

    The Boss room spawns the boss prefab.

    The Treasure room spawns the treasure item prefab.

## Troubleshooting

Nothing Generates When Pressing Play

    Ensure the DungeonGenerator component is attached to a GameObject.

    Confirm required prefabs are assigned in the Inspector:

        roomPrefab

        bossPrefab

    Make sure you opened the correct scene.

Doors Do Not Appear Correctly

    Ensure room prefab child objects are correctly named:

        Door_N, Door_E, Door_S, Door_W

        Wall_N, Wall_E, Wall_S, Wall_W

Enemies Do Not Spawn

    Ensure enemyPrefabs array is not empty.

Boss Does Not Spawn

    Ensure bossPrefab is assigned in the DungeonGenerator component.

    Confirm BossSpawner is functioning properly in the boss room.

## Notes

The dungeon layout is grid-based.

Boss placement is determined by BFS distance from the Start room.

Setting a non-zero seed allows deterministic dungeon layouts.
